x gt repo release ls ​

List repo release

  • Usage :
sh
x gt repo release ls [OPTIONS] [FLAGS]
x gt repo release ls [OPTIONS] [FLAGS]
  • Options :
Name, ShorthandDefaultDescription
--repo, -r <owner/repo><owner_path>/<repo_path>
--direction <asc,desc>"asc"list direction
  • Flags :
Name, ShorthandDescription
--json, -joutput raw JSON data

x gt repo release create ​

Create repo release

  • Usage :
sh
x gt repo release create [OPTIONS] [FLAGS]
x gt repo release create [OPTIONS] [FLAGS]
  • Options :
Name, ShorthandDefaultDescription
--repo, -r <owner/repo><owner_path>/<repo_path>
--tag <tag_name>Please provide tag name
--body <body_msg>Release Description(support markdown)
--target_commitish <branch|commit_SHA>branch name or commit SHA
--name <release_name>Release name(default:tag name)
  • Flags :
Name, ShorthandDescription
--prereleaseMark pre-release version
--json, -joutput raw JSON data
  • Examples :

Create release name as v1.1.0 release desc as '# v1.1.0' to main branch on <org>/<repo>

sh
x gt repo release create --repo <org>/<repo> --tag v1.1.0 --target_commitish main --body "# v1.1.0"
x gt repo release create --repo <org>/<repo> --tag v1.1.0 --target_commitish main --body "# v1.1.0"
